# Visual Design Note

This is a high-level aesthetic note, not the implementation contract.
If there is any conflict, `docs/guidelines/shell-design-system.md` wins for shell UI.

## Aesthetic Direction
- Modern industrial SaaS.
- Compact and content-first.
- Calm hierarchy over decoration.
- Neutral graphite shell surfaces.
- Restrained blue for action and selection.
- Cinematic dark for immersive shell contexts like Share and Login.

## Important Distinction
- Shell UI is the tool interface around the experience.
- VCA UI is the experience being designed.
- They should feel related in quality, but not identical in visual language.

## Use This Doc For
- High-level vibe alignment.
- Quick aesthetic reminders.

## Do Not Use This Doc For
- Detailed shell implementation rules.
- Token decisions.
- Component construction guidance.

For those, use:
`docs/guidelines/shell-design-system.md`
