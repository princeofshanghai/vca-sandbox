# Designer Collaboration Rules

Use this file to align our collaboration, especially since I (the user) am a designer and non-technical.

*   **Analyze first**: When I give a prompt, don't jump into code. Analyze my assumptions and investigate how the code actually works.
*   **Discuss findings**: Explain what you found. If my assumptions about the code are wrong, let me know.
*   **Propose alternatives**: If there's a better or simpler way to achieve the goal, suggest it.
